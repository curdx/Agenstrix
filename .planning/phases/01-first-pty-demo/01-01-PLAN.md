<objective>
Walking Skeleton for Agenstrix: prove the end-to-end vertical stack works with the smallest possible echo before any real `claude` lives behind the wire. This plan stands up the Bun runtime + Drizzle/SQLite + Hono HTTP/WS/SSE + React 19/Vite 8/Tailwind v4/shadcn-ui + xterm.js + bus + PTY abstraction (with Bun.Terminal-backed implementation + bun-pty fallback statically imported), the three-column chat shell with one MessageCard + embedded WorkerTerminal, plus the self-test/doctor scaffold and the `~/.agenstrix/` home directory. The Master "worker" is a placeholder PTY running `echo` (or `sleep`) that demonstrates the data path — Plan 02 swaps it for real `claude`.

Purpose: De-risk Bun.Terminal (Bun 1.3.14, 4 days old), the Hono `hono/bun` WS export pattern, the xterm.js v6 boot sequence with WebGL/Unicode11 addons, the Drizzle `bun-sqlite` migrator pattern, the shadcn v4 + Tailwind v4 CSS-first wiring, and the SQLite durability story — all in one plan so every later plan extends a working slice rather than scaffolding from zero.

Output: A repository that boots, opens a browser, shows a chat shell, persists bytes, and survives a tab refresh. No real `claude`, no kill-group, no redactor, no CI — those are Plans 02-06.
</objective>

**As a** developer using Agenstrix for the first time,
**I want to** run `bunx agenstrix` in any directory and immediately see a chat-shell in my browser with a live xterm-rendered terminal showing a real `claude` PTY (whose output is persisted to SQLite so closing the tab does not lose history, and whose process group is cleanly killed on shutdown with no orphans),
