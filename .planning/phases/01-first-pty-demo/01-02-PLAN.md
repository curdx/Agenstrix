<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Bun process → real `claude` subprocess | Now carrying the user's actual Anthropic credentials (read from `~/.claude/` by claude itself) — not in argv or env, but the spawned process can read them. Acceptable: same trust level as the user running `claude` directly in their own shell. |
| `claude` PTY bytes → SQLite | bytes may contain rendered output of the user's prompts (which could include secrets the user typed); redactor stub in Plan 01 catches none of these — SEC-01 fix lands in Plan 05 |
| WS heartbeat → client | Empty binary frames could be confused with PTY output; client (Pattern 4) explicitly checks `chunk.length === 0` → skip `term.write` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-02-01 | Information Disclosure | `claude` output containing API keys printed by tool calls | accept | Mitigation deferred to Plan 05 (SEC-01). Phase 1 acceptance: localhost-only, single-user; user is in control of their own claude session |
| T-01-02-02 | Tampering | Malformed JSON on WS triggering server-side parse-crash | mitigate | `try/catch` around `JSON.parse(data)` in `onMessage`; logged to pino, not thrown |
| T-01-02-03 | Denial of Service | Browser sending high-frequency resize events overwhelming PTY ioctl | mitigate | xterm's ResizeObserver fires only on layout change (debounced by browser); no additional throttle needed for single-user local |
| T-01-02-04 | Tampering | ANSI splitter losing bytes on shutdown (mid-sequence) | mitigate | `flushNow()` on SIGINT forces buffered bytes to flush; test 8 in batcher.test.ts asserts no infinite hold |
</threat_model>
